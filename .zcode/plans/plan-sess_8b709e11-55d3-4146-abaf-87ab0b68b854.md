## kern-web "signal everywhere" build — 8 features, one shared foundation

All 8 features share the same backbone (canvas loop, reduced-motion gating, global providers). I'll lay the shared infrastructure first, then layer each feature in dependency order. Everything is grounded in your existing patterns — `RadarShader`'s canvas recipe, `Reveal`/`SectionHeading`, the lowercase mono voice, the 7-hue semantic palette, `prefers-reduced-motion` (both the global CSS gate *and* `useReducedMotion()` from `motion/react`).

**Confirmed decisions:** signal strip *under* the nav (3px, `top:56px`); synthesized sound built now (Web Audio, zero asset files); all 8 in dependency order.

### Phase 0 — Shared foundation (built once, used everywhere)

**1. `hooks/useRafLoop.ts`** (new `hooks/` dir — none exists today)
Extract the 30fps-gate + delta-time block duplicated verbatim in `RadarShader.tsx:104-113` and `RegistryRadar.tsx`. Signature:
```ts
useRafLoop(callback: (t: number, dt: number) => void, opts?: { fps?: number })
```
- Handles: rAF schedule/cancel, the `last = now - (dt % frameInterval)` remainder-carry, `visibilitychange` pause + resume-with-reset (currently only in RadarShader), `pagehide`.
- Uses reactive `useReducedMotion()` (from `motion/react`) — standardizes on the reactive hook so toggling OS reduced-motion mid-session works (RadarShader's one-shot `matchMedia` read gets fixed for free).
- **Then refactor `RadarShader.tsx` and `RegistryRadar.tsx` to consume it** — removes ~40 lines of duplication and gives RegistryRadar the visibility-pause it's currently missing.

**2. `lib/canvas.ts`** — `setupCanvasDPR(canvas, ctx)` → `{ width, height, dpr }` (the `Math.min(devicePixelRatio||1,2)` + `setTransform` block, also duplicated). Plus a `ResizeObserver`-based helper so non-fullscreen canvases (the signal strip) re-size correctly.

**3. `hooks/useSweepProximity.ts`** — generalize `Hero.tsx`'s local `useSweepIllumination` (lines 21-64) to take an explicit radar center instead of assuming viewport center. Keeps the `radarsweep` window-event contract intact.

### Phase 1 — The "site feels alive" trio

**4. `components/layout/SignalStrip.tsx`** — the oscilloscope header bar.
- Thin canvas pinned `fixed inset-x-0 top-14 z-40 h-[3px]` (under the 56px nav; nav keeps `top-0 z-50`). Drawn on `useRafLoop` at ~30fps.
- Renders rhythmic pulses in the three semantic colors: mostly signal-green traveling micro-waves, occasional amber "transitional" blips, rare crimson "fault" flickers. Same palette vocabulary as `StatusDots`. `aria-hidden`, `pointer-events-none`.
- Listens for `radarsweep` events to sync a brightness pulse with the hero sweep (ties the whole site to one "heartbeat").
- Under reduced-motion: renders a single static faint green line (no rAF).
- Mounted once in `app/layout.tsx` inside `<AuthProvider>` as a sibling of `<StickyNav />`.

**5. `components/layout/BootSequence.tsx`** — the one-time power-on.
- `fixed inset-0 z-[90]` overlay (below the skip-link's `z-[100]`, above content). Self-unmounts.
- Gated by `sessionStorage["kern-booted"]` — renders once per browser session, never again.
- ~600ms choreography: core glow blooms (reuse `RadarMark`'s geometry), rings illuminate outward, then `opacity → 0` and unmount. Pure CSS keyframes (`@keyframes boot-power-on` in `globals.css`) + a `setTimeout` unmount.
- Under reduced-motion: skip entirely (set the flag, render nothing) — never makes a reduced-motion user wait.

**6. Cursor-reactive bloom** — `hooks/useSpotlight.ts` + a `@utility spotlight` in `globals.css`.
- Hook returns `onPointerMove` / `onPointerLeave` handlers that set `--bloom-x` / `--bloom-y` CSS vars on the element.
- The `spotlight` utility paints a `::before` radial green bloom (`rgba(76,245,160,…)`) positioned at those vars, opacity 0 → visible on hover, pointer-events-none. No global provider, opt-in per surface.
- Apply to cards: `PluginBrowser` cards, `PluginCards`, `DownloadSection` cards, `ReleaseCard`. Reduced-motion: bloom still allowed (it's pointer-driven, not autonomous) but the CSS global gate already neutralizes the transition.

### Phase 3 — Sound (synthesized, zero assets)

**9. `components/sound/SoundProvider.tsx`** + `hooks/useSound.ts` — Web Audio API, fully synthesized.
- **No audio files.** All voices are `OscillatorNode` + `GainNode` graphs built on first user gesture (browsers block `AudioContext` until then — resume on the first `pointerdown`/`keydown`).
- Voices: `hum` (continuous low drone: ~60Hz sine + filtered noise bed, very low gain), `blip` (short green-transition sine ping), `fault` (descending crimson sawtooth), `sweep` (low tone synced to the hero radar sweep via the `radarsweep` event).
- **Mute by default.** Toggle persisted in `localStorage["kern-sound"]`. A small speaker glyph toggle lives in `StickyNav` (next to auth). State exposed via `useSound()` → `{ enabled, toggle, play }`.
- **Silenced under reduced-motion** — the toggle renders disabled with a `// silenced (reduced motion)` tooltip; even if `localStorage` says on, no sound plays. This is the "prefers-reduced-motion-style silence default."
- Wired into existing surfaces: `TerminalMock` command echoes → `blip`; `LifecycleMock` start/stop → `blip`/lower blip; `error.tsx` → `fault`. Hero sweep tone gated to only fire when sound is on AND not reduced-motion.
